# Project 10: AI + MLOps — Production ML Platform with Full Observability

    Build a complete MLOps platform: a training pipeline (DVC-tracked data versioning, MLflow experiment tracking), automated retraining triggered by data drift detection (evidently library), a FastAPI model serving layer with canary deployment logic (route 10% traffic to new model, 90% to stable), Prometheus metrics exposed from the serving layer, a Grafana dashboard showing latency/accuracy/drift over time, and a GitHub Actions CI/CD pipeline that runs pytest unit tests, builds a Docker image, and deploys on pass. Format: WebApp + infrastructure (Docker Compose stack, CI/CD pipeline config, Grafana dashboard).

    Covers: File system automation (pathlib managing experiment artifacts, DVC cache), structured logging with Prometheus exposition (logger feeds metrics endpoint), try/except/finally at infrastructure level (graceful degradation if new model fails canary), virtual environments and dependency resolution deeply (poetry lockfiles, Docker layer caching, dependency conflict debugging), unit testing philosophy fully implemented (pytest fixtures, mock model objects, pipeline integration tests), process/resource management (Docker memory limits, model loading strategies to avoid OOM on small servers).

    Swiss value: Swiss engineering culture treats operational excellence as non-negotiable. This project speaks directly to MLOps Engineer roles at Swisscom, Swiss Post, and mid-size Zurich tech companies. It also demonstrates EU AI Act compliance readiness (audit trails, drift monitoring, versioned models) — a growing procurement requirement for Swiss enterprise software.
